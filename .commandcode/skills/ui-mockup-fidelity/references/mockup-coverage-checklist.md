# Mockup Coverage Checklist

Use this checklist before finalizing implementation.

## Coverage Inventory

List each visible item from the mockup and classify it:

- section wrapper
- layout region
- component (card, nav, form, table, CTA, etc.)
- interactive state (hover, active, focus, disabled)
- decorative/background layer

## Status Labels

Classify each item as one of:

- `implemented`
- `intentionally adapted`
- `unclear`

No item should remain unclassified.

## Fidelity Questions

- Is the section order identical?
- Does each major section use explicit background treatment?
- Is the 12-column grid baseline preserved where applicable?
- Are spacing and alignment relationships preserved?
- Are color roles and emphasis patterns preserved?
- Are stateful UI cues represented where visible?

## Final Review Output

Return these sections in your final response:

- `What matches exactly`
- `What was adapted`
- `What needs clarification`
